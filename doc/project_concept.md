## 1. 프로젝트 개요

### 1.1 프로젝트명
**쉽계부(TopDownBudget)** - 약간의 SNS 기능을 포함한 처음 가계부를 쓰는 사람을 위한 쉬운 가계부

### 1.2 프로젝트 목적
기존의 가계부는 Bottom Up 형태로 내역을 한건씩 등록하던 과정때문에 가계부를 시작하는 것도 어려웠기에 Top Down 형태로 관리하고 혼자 고민하면 어디서 어떻게 관리할지, 어떻게 아낄지 막막해 가계부 작성을 포기했던 사람들을 위한 SNS 기능이 포함된 가계부

### 1.3 핵심 가치
- **프라이버시**: 공유를 선택하지 않은 기본적인 가계부 기능은 Private 형태로 원하는 사람한테만 공유하거나, 아예 혼자만 사용할 수 있음
- **편의성**: 직관적인 UI/UX와 전체부터 상세 관리로 이어지는 쉬운 가계부 작성(Top Down 형태 기록으로 기존 가계부에서 Bottom Up으로 취합되는 것과 달리 전체 수입, 지출 -> 통장, 카드 별 지출 -> 상세 지출 형태로 관리하고싶은 수준까지만 관리할 수 있음)
- **SNS 기능**: 원하는 항목이나 달의 내역을 SNS 형태로 이번 달 지출을 자랑하거나, 알뜰 소비 자랑, 알뜰하게 사는 팁을 공유하고, 가계부 쉽게 잘 관리하는 팁도 공유하는  사용하는 SNS적인 기능
- **빠른 검색**: 분류, 내용, 태그, 작성일 기반 검색 기능

### 1.4 기술 스택

#### 프론트엔드
- **Framework**: Next.js 16.1.1
- **스타일링**: Tailwind CSS 4
- **UI 컴포넌트**: shadcn/ui
- **에디터**: Tiptap
- **차트**: Chart.js + react-chartjs-2

#### 백엔드
- **서버**: Next.js Server Actions
- **인증**: Supabase Auth
- **데이터베이스**: Supabase PostgreSQL
- **스토리지**: Supabase Storage (이미지??추후 고도화 설계 예정)



#### 기타
- **언어**: TypeScript
- **날짜 처리**: 한국 시간(KST) 기준

### 2.2 라우팅 구조
```
/                    # 메인 홈페이지 (가계부 목록, 피드...)
/diary/[id]          # 가계부 id 별 상세 페이지(public은 누구나, private은 권한 있는 사람만 접근)
/diary/[id]/edit     # 가계부 수정 페이지(가계부 수정 권한이 있는 사람만 접근 가능, owner 권한 있는 사람은 권한 수정도 가능)
/new                 # 새 가계부 추가 페이지
```

#### 라우팅 설계 원칙 (첨언)
- 한 계정이 여러 가계부를 동시에 운영할 수 있도록 설계한다.
- 가계부를 분리하는 주요 목적은 구독 가계부, 공통 관리 가계부(계비 등), 가족 가계부, 개인 용돈 가계부를 독립적으로 관리하기 위함이다.
- 메인 페이지(`/`)에서 사용자가 접근 가능한 가계부를 카드 목록으로 보여주고, 선택한 카드의 `id`로 상세 페이지(`/diary/[id]`)에 진입한다.
- 상위 라우팅을 계정 단위로 추가 분리하지 않고, 메인 카드 노출 로직과 가계부별 권한 검사로 접근을 제어한다.

### 2.3 가계부 유형 및 권한 모델

#### 가계부 유형 정의
- **public**: 누구나 검색해서 상세 페이지를 열람할 수 있는 공개 가계부.
- **private**: 참여자로 등록된 사용자만 접근 가능한 비공개 가계부.

#### 권한 역할 정의
- **owner**: 가계부 생성자. 가계부 삭제, 참여자 권한 변경, 공개 범위 변경 등 최고 권한.
- **manager**: 공동 관리 권한. 예산/지출 수정, 카테고리 관리, 참여자 초대 가능(권한 승급/강등 불가).
- **viewer**: 조회 권한. private 가계부에서만 사용하는 읽기 전용 참여자 역할.

#### 기능별 권한 범위
| 기능 | owner | manager | viewer |
|---|---|---|---|
| private 가계부 상세 조회 | O | O | O |
| 수입/지출 등록 | O | O | X |
| 수입/지출 수정/삭제 | O | O | X |
| 예산(월/카테고리) 설정 | O | O | X |
| 태그/카테고리 관리 | O | O | X |
| 참여자 초대 | O | O | X |
| 참여자 권한 변경 | O | X | X |
| 가계부 공개 범위 변경 | O | X | X |
| 가계부 삭제 | O | X | X |

#### 유형별 열람 규칙
| 가계부 유형 | 비참여 사용자 | viewer | manager | owner |
|---|---|---|---|---|
| public | O (검색/상세 열람 가능) | 불필요 | O | O |
| private | X | O | O | O |

#### 페이지 접근 규칙
- `/` : 로그인 사용자는 자신이 접근 가능한 private 가계부 카드와 public 가계부 카드를 함께 본다.
- `/diary/[id]` : public은 누구나 접근 가능, private은 owner/manager/viewer만 접근 가능.
- `/diary/[id]/edit` : owner 또는 manager만 접근 가능.
- `/new` : 로그인 사용자면 접근 가능, 생성 즉시 owner 권한 부여.

#### 정책 상세
- 사용자-가계부 관계는 역할 단위(owner/manager/viewer)로 저장한다.
- 한 사용자는 가계부마다 다른 역할을 가질 수 있다. 예: 가족 가계부는 manager, 개인 용돈 가계부는 owner.
- public 가계부는 조회 목적의 viewer 참여자 등록 없이도 열람 가능하다.
- private 가계부는 참여자 등록이 필수이며, 미등록 사용자는 상세 정보에 접근할 수 없다.
- 접근 제어는 UI 표시 제어와 서버 권한 검사(Supabase RLS + Server Actions) 둘 다 적용한다.
- RLS 조회 정책은 `is_public = true` 이거나 해당 가계부의 참여자인 경우에만 허용한다.

## 3. 기능 요구 사항

### 3.1 테마(다크모드) 기능
- 전역 테마 토글(라이트/다크/시스템 연동)을 제공한다.
- 토글 위치는 상단 네비게이션(헤더) 우측 고정 영역을 기본으로 한다.
- 사용자가 선택한 테마는 다음 접속 시에도 유지되도록 저장한다.

### 3.2 가계부 상세 화면 사이드바
- 가계부 상세(`/diary/[id]`)에 고정형 사이드바를 제공한다.
- 사이드바에는 다음 기능을 포함한다.
    - 월/일 이동 가능한 달력 필터
    - 통합 검색 입력(내용, 메모, 태그)
    - 분류별(카테고리) 필터
    - 일자 범위 필터(시작일~종료일)
- 필터 상태는 URL 쿼리 또는 상태값으로 유지해 새로고침 후에도 동일 조회가 가능해야 한다.

### 3.3 내역 검색/조회 기능
- 사용자는 분류별, 일자별, 키워드 기반으로 지출/수입 내역을 검색할 수 있어야 한다.
- 복수 조건(예: 식비 + 7월 1일~7월 15일 + "점심") 동시 검색을 지원한다.
- 검색 결과는 최신일 우선 정렬을 기본값으로 한다.

### 3.4 통계 시각화 기능
- 가계부 메인 영역(상세 진입 후 기본 화면)에 요약 통계 그래프를 제공한다.
- 막대그래프:
    - 일별 또는 월별 흐름(수입/지출/순지출)을 보여준다.
    - 사용자는 일별/월별 집계 단위를 전환할 수 있어야 한다.
- 원형그래프:
    - 현재 조회 월의 카테고리별 지출 비율을 보여준다.
    - 항목 hover 또는 선택 시 금액/비율 툴팁을 제공한다.

### 3.5 권한 연계 요구사항
- public 가계부는 비참여 사용자도 상세 및 통계 열람 가능하다.
- private 가계부는 owner/manager/viewer만 열람 가능하며, 미참여자는 접근할 수 없다.
- 수정 가능한 통계 조건 저장, 필터 저장 등 사용자별 설정은 로그인 사용자 기준으로 분리한다.

### 3.6 통화 정책 (MVP)
- 서비스 기본 통화는 KRW로 한다.
- 가계부 생성 시 기본 통화(`base_currency`)를 지정하며 기본값은 KRW이다.
- 항목(수입/지출) 추가 시 통화 입력 기본값은 해당 가계부의 기본 통화를 자동 선택한다.
- 사용자는 항목별로 통화를 수동 변경할 수 있다(예: USD, JPY).
- MVP에서는 자동 환율 연동 및 실시간 환산을 제공하지 않는다.
- 통계(막대/원형)는 가계부 기본 통화 기준 집계를 우선 제공한다.

### 3.7 차트 라이브러리 선정 기준 및 결정
- 선정 기준
    - 범용성: 커뮤니티/레퍼런스가 충분히 많아야 한다.
    - 비용: 개인 프로젝트 무자본 시작 기준으로 무료 사용 가능해야 한다.
    - 유지보수성: React + TypeScript 조합에서 구현 난이도가 과도하지 않아야 한다.
- 후보 비교
    - Recharts: React 친화적이고 단순 대시보드 구현이 쉽다.
    - Chart.js(react-chartjs-2): 대중성과 문서/예제가 매우 풍부하고 생태계가 크다.
    - Apache ECharts: 기능은 강력하지만 초기 설정/커스텀이 상대적으로 무겁다.
- 최종 결정
    - MVP 기본 차트 라이브러리는 **Chart.js + react-chartjs-2**로 채택한다.
    - 사유: 범용성, 학습자료, 무상 사용, 유지보수 균형이 가장 적합함.

### 3.8 메인 화면 노출 정책
- 메인 화면 상단에는 로그인 버튼을 항상 노출한다(로그인 전 사용자 기준).
- 비로그인 상태에서는 public 가계부 중 구독자 수 20명 이상인 가계부를 대상으로 노출 후보를 구성한다.
- 후보 가계부는 구독자 수 내림차순으로 정렬하고, 상위 N개(초기 6개)를 카드로 노출한다.
- 조건을 만족하는 가계부가 없으면 빈 상태 안내 문구를 노출한다.
    - 예시 문구: "아직 인기 public 가계부가 없어요. 첫 번째 유명 가계부의 주인공이 되어보세요."
- 로그인 상태에서는 사용자가 접근 가능한 가계부(private 멤버십 + public)를 우선 노출하고, 추천 public 가계부를 보조 영역에 노출한다.

### 3.9 메인 Cockpit 요구사항
- 메인 화면 카드 영역 상단에 요약 차트 패널(간략 막대/원형)을 배치한다.
- 사용자가 조회 중인 가계부를 바꾸는 방식은 2가지 중 하나를 제공한다.
    - 카드 슬라이드(캐러셀)로 활성 가계부 전환
    - 콤보박스(셀렉트)로 활성 가계부 선택
- 활성 가계부가 변경되면 요약 차트 패널과 카드 상세 정보가 함께 갱신되어야 한다.
- 요약 패널은 상세 화면의 전체 통계를 대체하지 않고, 빠른 비교/탐색용으로만 사용한다.

### 3.10 하이라키(피벗형) 가계부 구조
- 가계부 상세 내역은 엑셀 피벗 테이블처럼 트리(계층) 구조로 관리한다.
- 상위 항목(부모)과 하위 항목(자식) 모두 금액 입력이 가능해야 한다.
- 사용자는 항목 접기/펼치기로 단계별 합계를 확인할 수 있어야 한다.
- 트리 그리드는 최소 3단계(대분류/중분류/소분류)를 지원하고, 필요 시 확장 가능하도록 설계한다.

### 3.11 자동 요약 항목(기타) 규칙
- 상위 항목에는 사용자가 직접 입력한 상위 금액(`parent_amount`)을 둘 수 있다.
- 하위 항목 합계(`children_sum`)를 기준으로 잔액 항목인 `기타`를 자동 계산한다.
- 계산식: `기타 = parent_amount - children_sum`
- `기타`는 시스템 자동 계산값이며, 사용자가 직접 수정/삭제할 수 없다.
- 하위 항목을 추가/수정/삭제하면 `기타` 값은 즉시 재계산되어 summary에 반영된다.
- `children_sum`이 `parent_amount`를 초과하면 `기타`는 음수로 표시하고 초과 상태 배지를 노출한다.

### 3.12 엑셀 복사/붙여넣기 입력
- 사용자는 엑셀에서 복사한 표 데이터를 가계부 상세 입력 영역에 붙여넣을 수 있어야 한다.
- 붙여넣기 데이터는 탭/줄바꿈 기반 TSV 형식을 기본으로 파싱한다.
- 파싱 가능한 기본 컬럼: 일자, 분류(계층 경로), 금액, 메모, 통화(선택).
- 유효성 검사 실패 행은 전체 실패가 아닌 행 단위 오류로 표시하고, 정상 행은 우선 반영한다.
- 붙여넣기 완료 후 트리 합계와 `기타` 자동 계산을 즉시 갱신한다.

### 3.13 하이라키 트리 편집 권한 규칙
- owner, manager는 트리 항목(분류/금액/메모) 생성/수정/삭제가 가능하다.
- viewer는 트리 조회만 가능하며, 편집 UI를 노출하지 않는다.
- `기타(자동)` 행은 owner/manager도 직접 편집할 수 없다(시스템 계산 전용).
- public 가계부라도 비참여 사용자는 조회만 가능하고 편집은 불가하다.
- 분류 구조 변경(부모 이동/순서 변경)은 owner, manager만 가능하며, 대량 구조 변경 기능은 owner 전용으로 제한할 수 있다.

## 4. UI/UX 설계

### 4.1 레이아웃 개요
- 헤더: 서비스 로고, 로그인 버튼(비로그인 시), 다크모드 토글, 사용자 메뉴(로그인 시).
- 메인 랜딩 본문: Cockpit 요약 패널 + 가계부 카드 영역 + 추천/안내 영역.
- 가계부 상세 본문: 좌측 사이드바(달력/검색/필터) + 우측 메인 패널(통계 + 내역 리스트).
- 모바일: 카드/차트는 세로 스택으로 재배치하고, 상세의 사이드바는 Drawer 형태로 전환한다.

### 4.2 메인 화면 상세 레이아웃 (Landing + Cockpit)
- 섹션 A (Top Bar)
    - 좌측: 로고
    - 우측: 로그인 버튼 또는 사용자 아바타 메뉴
- 섹션 B (Cockpit 패널)
    - 활성 가계부 선택 컨트롤: 캐러셀 또는 콤보박스
    - 간략 막대그래프: 최근 흐름(일별/월별 중 1개 기본값)
    - 간략 원형그래프: 조회 월 카테고리 비율
- 섹션 C (가계부 카드 그리드)
    - 카드 정보: 가계부명, 공개 유형(public/private), 구독자 수, 최근 활동일
    - 카드 액션: 상세 보기, 구독(또는 즐겨찾기)
- 섹션 D (빈 상태)
    - 조건: 노출 가능한 인기 public 가계부가 없을 때
    - 메시지 + CTA 버튼(예: "첫 가계부 만들기")

### 4.3 가계부 상세 화면 구성
- 상단 요약 카드: 이번 달 예산, 사용 금액, 잔여 금액, 초과 여부.
- 통계 영역:
    - 좌측: 일별/월별 흐름 막대그래프
    - 우측: 월별 항목 비율 원형그래프
- 내역 영역: 피벗형 트리 그리드(접기/펼치기) + 정렬 옵션 + 페이징(또는 무한 스크롤).
- 트리 그리드 행 구성: 항목명, 상위 입력 금액, 하위 합계, 기타(자동 계산), 실제 집행액, 잔액.
- 트리 그리드 상단에 `엑셀 붙여넣기` 입력 모드를 제공한다.

### 4.4 인터랙션 원칙
- 필터 변경 시 통계 그래프와 내역 리스트가 동시에 재조회되어 일관된 결과를 보여준다.
- 그래프 클릭 시 해당 구간/카테고리로 내역 리스트를 드릴다운한다.
- 검색/필터가 적용된 상태를 화면 상단에 칩 형태로 노출해 해제하기 쉽게 한다.
- 메인 Cockpit에서 활성 가계부를 변경하면 카드 포커스와 요약 차트가 동시에 변경된다.
- 하위 항목 편집 시 상위 summary와 `기타` 값은 즉시 재계산되어 화면에 반영된다.
- `기타` 셀은 읽기 전용 스타일(비활성/잠금 아이콘)로 표시한다.
- 엑셀 붙여넣기 후 처리 결과(성공 N건, 실패 N건, 실패 사유)를 토스트/패널로 제공한다.

### 4.5 접근성 및 가독성
- 다크모드에서도 색상 대비를 유지하고, 그래프 색상은 색각 이상 사용자를 고려한 팔레트를 사용한다.
- 숫자 표기는 천 단위 구분자와 통화 단위를 일관 적용한다.
- 날짜 표기는 KST 기준으로 통일하고, 상대 시간 대신 명시 날짜를 기본으로 제공한다.

### 4.6 통화/그래프 UI 세부 원칙
- 통화 표시는 기본적으로 가계부 기본 통화(KRW)를 우선 사용한다.
- 항목 통화가 기본 통화와 다를 경우 내역 리스트에서 통화 배지를 표시한다.
- 그래프 영역 우측 상단에 집계 기준(일별/월별) 토글을 배치한다.
- 원형그래프는 조회 월 기준 카테고리 비율만 표시하고, 범례 클릭으로 카테고리 on/off 필터를 제공한다.

### 4.8 하이라키 트리 그리드 표시 규칙
- 분류 컬럼은 들여쓰기와 토글 버튼으로 계층을 표현한다.
- 각 부모 행 하단에 `기타(자동)` 가상 행을 노출한다(사용자 편집 불가).
- `기타(자동)`는 부모 레벨 summary의 일부로만 집계되며, 독립 카테고리로 생성하지 않는다.
- 음수 기타가 발생한 부모 행은 경고 색상 및 초과 아이콘으로 표시한다.
- owner/manager에게만 셀 편집, 행 추가, 행 삭제 액션 버튼을 노출한다.
- viewer 및 비참여 사용자는 읽기 전용 그리드로 노출한다.

### 5.3 메인 노출용 집계 기준
- 인기 public 가계부 집계는 diary별 구독자 수(`count(diary_subscriptions.user_id)`)를 기준으로 계산한다.
- 비로그인 메인 노출 조건은 다음과 같다.
    - `is_public = true`
    - 구독자 수 >= 20
    - 구독자 수 내림차순 정렬 후 상위 N개 조회

### 5.4 RLS 개요
- diaries/entries/categories 조회: `diaries.is_public = true` 또는 멤버십 존재 시 허용.
- 수정/삭제: owner 또는 manager만 허용.
- 권한(role) 변경/가계부 삭제: owner만 허용.

### 5.5 하이라키 편집 권한 상세(RLS/서버)
- categories/entries insert, update, delete: owner/manager만 허용.
- diary_memberships update/delete: owner만 허용.
- `기타`는 계산 뷰에서만 제공하므로 직접 update/delete 경로를 두지 않는다.
- Server Actions에서도 동일 권한 검사를 수행해 UI 우회 요청을 차단한다.

### 5.6 통화 데이터 처리 원칙
- `diaries.base_currency`는 가계부 기본 통화를 의미한다.
- `entries.currency`는 항목 입력 통화를 의미하며 기본값은 `diaries.base_currency`를 따른다.
- 자동 환율 변환은 추후 고도화 항목으로 분리한다.

### 5.7 하이라키/기타 계산 데이터 원칙
- 상위 카테고리 입력값은 `categories.parent_amount`에 저장한다.
- 하위 합계(`children_sum`)는 조회 시점 집계값으로 계산한다.
- `기타`는 저장 컬럼이 아닌 계산 필드(뷰/쿼리 결과)로 제공한다.
- 계산식: `other_amount = parent_amount - children_sum`
- `기타`는 가상 행이므로 직접 insert/update/delete 대상이 아니다.

### 5.8 기타 자동계산 SQL 뷰 초안
```sql
create or replace view v_category_summary_with_other as
with child_sum as (
    select
        c.parent_category_id,
        sum(e.amount) filter (where e.entry_type = 'expense') as children_sum
    from categories c
    left join entries e on e.category_id = c.id
    group by c.parent_category_id
)
select
    p.id as category_id,
    p.diary_id,
    p.name,
    p.parent_amount,
    coalesce(cs.children_sum, 0) as children_sum,
    coalesce(p.parent_amount, 0) - coalesce(cs.children_sum, 0) as other_amount,
    case
        when coalesce(p.parent_amount, 0) - coalesce(cs.children_sum, 0) < 0 then true
        else false
    end as is_over_allocated
from categories p
left join child_sum cs on cs.parent_category_id = p.id;
```

뷰 활용 원칙:
- UI는 `other_amount`를 `기타(자동)`로만 노출한다.
- `other_amount`는 쓰기 대상이 아니며, 상위 금액 또는 하위 내역 변경으로만 변한다.
- 월 단위 통계가 필요하면 `entry_date` 필터를 받는 materialized view 또는 RPC로 확장한다.

### 5.9 엑셀 붙여넣기 처리 원칙
- 붙여넣기 API는 배치 단위로 입력값을 검증한 뒤 행 단위 upsert/insert를 수행한다.
- 실패 행은 오류 코드(날짜 형식, 금액 형식, 분류 경로 미존재 등)와 함께 반환한다.
- 성공 반영 후 관련 부모 카테고리 summary 집계를 재조회한다.
